## Objective
Create the architecture baseline artifacts that define module contracts, allowed dependency direction, and spec ownership boundaries so later implementation cannot violate engine-first and deterministic constraints.

## Constraints
- This plan is documentation-only.
- No implementation code should be added in this phase.
- No UI/editor scope is allowed.

<tasks>
  <task id="01-01-01">Create `docs/architecture/07-module-contract-matrix.md` with per-module responsibilities, input/output contracts, and forbidden couplings.</task>
  <task id="01-01-02">Create `docs/architecture/08-dependency-boundaries-adr.md` defining allowed dependency direction (`Core -> Engine -> Renderer/Export -> CLI` as applicable) and anti-leakage rules.</task>
  <task id="01-01-03">Create `docs/architecture/09-spec-ownership-map.md` mapping top-level spec sections (`meta`, `assets`, `scene`, `timeline`, `output`) to owning module contracts.</task>
  <task id="01-01-04">Add a short architecture review checklist section in each new doc to enforce "no hardcoded scene logic" and deterministic behavior expectations.</task>
  <task id="01-01-05">Cross-check terminology and constraints against `AGENTS.md`, `.planning/ROADMAP.md`, and `.planning/REQUIREMENTS.md` for consistency before completion.</task>
</tasks>
