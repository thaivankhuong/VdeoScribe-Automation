## Objective
Introduce one Engine-owned render-sequence planner and replace single-frame CLI execution with deterministic full-run orchestration for both run and batch flows.

## Constraints
- Engine owns project-duration and frame-range derivation; CLI only consumes the ordered plan.
- Do not change existing timeline, draw, camera, or object semantics per frame.
- Keep execution deterministic and serial by default in this phase.
- Do not introduce playable encoding or audio muxing; Phase 07 still ends at frame artifact generation and artifact-ready packaging.
- If single-frame execution is preserved, it must be explicitly secondary/debug-only rather than the primary run semantics.
