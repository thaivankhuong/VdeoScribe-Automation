## Objective
Implement deterministic per-frame object lifecycle resolution using ordered timeline inputs and prior state so frame-state outputs become a stable contract for downstream rendering/export phases.

## Constraints
- Lifecycle resolution must consume canonical normalized inputs and ordered timeline outputs from `02-01` and `02-02`.
- No renderer/export/CLI business logic additions; CLI may only orchestrate and validate outputs.
- Maintain strict deterministic behavior for object ordering and transition precedence.
- Keep behavior aligned to Phase 02 core only; draw progression details beyond lifecycle state remain for Phase 03.
