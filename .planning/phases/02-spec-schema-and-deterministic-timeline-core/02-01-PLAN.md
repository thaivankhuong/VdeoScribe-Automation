## Objective
Implement a deterministic spec processing pipeline in Core that validates and normalizes project JSON into canonical inputs consumed by Engine resolvers, with CLI loader integration that fails fast on ordered validation output.

## Constraints
- Stay within Phase 02 scope: schema, normalization, and deterministic timeline core only.
- No UI/editor behavior, rendering behavior, or export logic.
- Preserve module boundaries: validation/normalization in Core, orchestration in CLI, timeline evaluation in Engine.
- Deterministic behavior is mandatory; no reliance on runtime dictionary/list incidental ordering.
