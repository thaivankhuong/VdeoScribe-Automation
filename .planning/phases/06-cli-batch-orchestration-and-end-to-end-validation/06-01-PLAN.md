## Objective
Implement the Phase 06 CLI command surface, manifest-driven batch request/result contracts, a dedicated CLI parser seam, a persisted `--summary-output <path>` JSON summary artifact, and deterministic orchestration flow needed to run repeatable single-job and multi-job pipeline executions.

## Constraints
- Keep CLI orchestration-only; do not reopen engine, renderer, or export semantics.
- Preserve `PipelineOrchestrator.Run(CliRunRequest)` as the authoritative single-job end-to-end seam.
- Normalize batch ordering explicitly by canonical `jobId`; never rely on file-system enumeration order or incidental JSON property ordering.
- Detect duplicate normalized `jobId` values deterministically as validation failures.
- Batch mode requires `--summary-output <path>` and always writes the JSON artifact for both success and failure cases.
- The JSON artifact must use stable camelCase field names, logical relative paths, explicit nulls, and stable array ordering by canonical `jobId`.
- Keep execution serial for deterministic Phase 06 behavior; no distributed execution, worker queues, or UI scope.
- Update the CLI test project wiring whenever new CLI source files or CLI-side test files are introduced, because `Whiteboard.Cli.Tests.csproj` disables default compile items.
- You are not alone in the codebase; do not revert unrelated edits while adding Phase 06 contracts.
