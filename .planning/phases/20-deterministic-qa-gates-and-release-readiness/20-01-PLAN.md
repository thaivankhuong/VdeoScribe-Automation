## Objective

Add a deterministic QA gate stage to batch jobs so compile/run success is not enough: each job must also match a committed regression baseline.

## Tasks

1. Extend batch manifest/job contracts with deterministic QA gate configuration (`enforceDeterministicQaGates`, default/per-job baseline paths).
2. Extend per-job and summary result contracts with gate fields (status, report path, gate deterministic key).
3. Add gate evaluation in `BatchPipelineOrchestrator` against baseline checks:
   - Project ID
   - Exported frame count
   - Exported audio cue count
   - Total duration
   - Optional anchor deterministic keys
4. Fail job deterministically on gate drift with `FailureStage = Gate`, no hidden retry.
5. Emit deterministic `qa-gate-report.json` artifacts per job attempt.
6. Add unit and integration coverage for:
   - Gate pass
   - Gate drift failure
   - No retry on gate failure
