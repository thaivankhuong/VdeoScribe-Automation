## Objective
Add the auditable reporting and diagnostics surface for the script compiler.

Purpose: satisfy `CMP-02` by making every compile produce a separate deterministic report artifact and by locking unsupported mappings, missing governed IDs, and downstream spec-validation failures behind stable diagnostic contracts that operators can act on.
Output: Core report/diagnostic records, compile-result/report serialization, CLI report output, failure fixtures, and deterministic reporting tests.
